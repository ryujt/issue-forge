# 코드 구조 및 스타일 가이드라인

## 가독성 우선

* 누구나 빠르게 읽고 이해할 수 있도록 단순하고 명확하게 작성한다.
* 불필요한 축약, 과도한 한 줄 처리, 복잡한 표현을 피한다.
* 함수와 메서드는 의도가 드러나는 이름을 사용한다.

## 단일 책임

* 하나의 파일은 하나의 책임만 가진다.
* 서로 다른 역할이나 단계별 로직은 별도 파일로 분리한다.
* 유지보수와 교체 범위가 최소화되도록 설계한다.

## 탑다운 구조

* 메인 엔트리 파일은 전체 흐름만 보여준다.
* 구체 로직과 구현 세부 사항은 별도 모듈, 클래스, 함수로 내려 보낸다.
* 상위 레벨 코드에서는 "무엇을 한다"만 보이고 "어떻게 한다"는 하위 구현으로 숨긴다.

## 상수 관리

* 매직 넘버와 매직 스트링은 직접 쓰지 않는다.
* 자주 사용하는 값은 상수로 선언한다.
* 상수 정의는 파일의 앞부분에 모아 둔다.

## 주석 금지

* 코드에 주석을 작성하지 않는다.
* 설명이 필요할 정도로 복잡한 코드는 구조를 다시 분리하거나 이름을 수정한다.
